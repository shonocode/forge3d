FORGE3D 操作マニュアル

ブラウザで動く Babylon.js ベースの 3D DCC ツール (Blender 的ポジション)。 このページは forge3d の機能と操作を一覧する一次資料。
キャラクター制作を実際にやってみたい場合は → くりまんじゅう制作チュートリアル

1. はじめに

Forge3D は モデリング / スカルプト / リギング / アニメ / エクスポート を ブラウザだけで完結させるツール。Blender に近い構成だが、Web 前提なので オフライン保存・PWA・タッチ対応が組み込まれている。

1.1 起動と画面構成

  • ヘッダー（上）: モード表示 (OBJECT / EDIT / SCULPT ...)、ツール pill、FPS・頂点数
  • 左パネル: プリミティブ、ツール (Sculpt/Paint/Bone/Weight/Anim)、CSG、メッシュユーティリティ
  • ビューポート（中央）: 3D 編集領域、Arc Rotate カメラ
  • 右パネル: タブ式の詳細設定 (Transform / Material / Edit / Anim ...)
  • ステータスバー（下）: 操作結果のメッセージ表示

1.2 保存と復元

  • 自動保存: 30 秒ごとに OPFS / IndexedDB にチェックポイント保存。クラッシュ時は次回起動で復元を提案
  • Model Library: Map タブの「Save to Library」で複数モデルをローカル保存
  • エクスポート: GLB / OBJ / STL ファイルとしてダウンロード
Map タブでライブラリのモデルを複数配置する場合、2 体目以降は GPU インスタンス （ジオメトリ共有）として置かれる — 100 体置いてもメモリは 1 体分 + 変換行列。ステータスに 「Placed (instance)」と出た配置は移動/回転/スケールのみ可能（形状の個別編集は不可、 編集したい場合は最初の 1 体を使う）。

2. 基本操作

2.1 カメラ

  • 左ドラッグ: 回転 (Arc Rotate)
  • 中ドラッグ / ホイール押し: パン
  • ホイール / ピンチ: ズーム
  • テンキー 1/3/7: 前/右/上ビュー (Ctrl 併用で反対側)
  • テンキー 5: 透視/正投影トグル
  • F キー: 選択メッシュへフォーカス
  • ナビゲーションギズモ（ビューポート右上）: 現在のカメラから見た ±X/±Y/±Z の向きを常時表示。軸の丸をクリック/タップでそのビューへスナップ （テンキーのないタブレットでのビュー切替に）

2.1.1 リファレンス画像（下絵）

Scene タブの 🖼 Import Reference でコンセプトアートや三面図を 非ライティングの半透明プレーンとしてシーンに置ける。通常のメッシュ同様 Gizmo で移動・回転・スケールでき、レイヤーにも入る。GLB / OBJ / Save to Library には 一切含まれない（モデリング用の下絵であってアセットではない）。不要になったら普通に削除。

設定の永続化: ブラシ設定（Sculpt/Paint/Weight）・Auto-Key・シェーディング モード・環境プリセットは 自動でブラウザに保存され、次回起動時に復元される。

2.2 選択 / Gizmo

  • クリック: メッシュ選択
  • Ctrl/Cmd+クリック: 複数選択 (追加 / 解除)
  • V: Select ツール
  • G / R / S: 移動 (Move) / 回転 (Rotate) / 拡縮 (Scale) Gizmo
  • Ctrl+A: 全選択
  • Esc: 選択解除
Gizmo はドラッグした分だけ history に1エントリ。Ctrl+Z で 1 操作戻る。

2.2.1 Hierarchy パネル（アウトライナ）

  • 🔍 filter: 名前でメッシュを絞り込み（マッチをフラット表示、階層下のマッチも拾う）
  • 👁 ボタン: メッシュ単位の表示/非表示トグル（非表示の行は薄く表示）
  • ◎ Isolate: 選択メッシュだけを表示。もう一度押すと元の表示状態に復帰 （ビュー状態なので Undo 履歴には積まれない）。密集シーンでの部分作業に

2.3 Undo / Redo

  • Ctrl+Z: 元に戻す (最大 50 履歴)
  • Ctrl+Shift+Z: やり直し

Sculpt / Paint / Weight Paint のブラシストロークは「ストローク開始 → 終了」を 1 履歴として記録。

4. Edit Mode

頂点 / 辺 / 面を直接編集するモード。Blender の Tab で切り替わる Edit Mode に相当。形状の細部調整、ループ追加、コーナー処理など。

4.1 Edit Mode への入り方

  1. 編集したいメッシュを 1 つ選択
  2. Tab を押す → 右パネルが Edit タブに自動切替、頂点 dots が表示される
  3. もう一度 Tab で Object Mode に戻る

Edit Mode 中は他のツール (Sculpt/Paint 等) には切り替わらない。先に Tab で抜ける必要がある。

4.2 頂点 / 辺 / 面の選択

キーモード選択するもの
1Vertex頂点（白いドット）
2Edge辺（灰色のライン → 選択で黄）
3Face面（選択で半透明黄ハイライト）
  • クリック: 単一選択（既存の選択をクリア）
  • Ctrl+クリック: トグル選択（追加 / 解除）
  • B: Box Select → 次のドラッグで矩形選択
  • A: 全選択
  • Esc: 選択クリア

選択した要素の重心に Gizmo が現れる。ドラッグで選択要素全体を変形する。

4.2.1 コンポーネントの Move / Rotate / Scale

Gizmo は 3 種類あり、右パネル「Transform」の Move / Rotate / Scale ボタン、 またはキーで切り替える: T = Move、R = Rotate、S = Scale。

  • Move: 選択要素を平行移動（従来動作）
  • Rotate: 選択要素をドラッグ開始時の重心まわりに回転。 面の傾き調整、耳や腕の角度付けに
  • Scale: 重心基準で軸別スケール（中央の箱で均等スケール）。 リング状の選択をすぼめる / 広げる操作に

1 ドラッグ = 1 Undo。どのモードでも辺 / 面選択はその構成頂点がまとめて動く。

スナップ連動: Object Mode の Transform タブ「Snap」設定 (Position / Rotation / Scale の刻み) は Edit Mode のコンポーネントギズモにも適用される。 0.5m グリッドに頂点を吸着させながら移動、15° 刻みで面を回転、など。

4.2.2 プロポーショナル編集 (◉ Proportional)

Transform セクションの ◉ Proportional をオンにすると、選択まわり Prop. Radius (ローカル単位) 内の未選択頂点も、距離に応じて減衰しながら 一緒に変形する (Blender の O = soft select 相当)。Move / Rotate / Scale すべてに効く。

  • 減衰はスムーズステップ (選択位置で 100% → 半径の縁で 0%)
  • 回転は角度を、スケールは倍率を、減衰率でブレンド — 面がなめらかに繋がる
  • 有機的な形の調整 (頬をふくらませる、体の一部を引っ張る) は Sculpt の Grab と使い分け: 正確な頂点単位なら Proportional、勢いで盛るなら Sculpt

4.3 Extrude (押し出し)

選択を複製して新しい geometry を生やす。キー: E

Face モード — 面の押し出し

典型的な「指を生やす」「キューブの一面を引き出す」操作：

  1. Face モード（3）で面を 1 つ以上選択
  2. E → 選択面の cap が複製される（位置は同じ、重なって見える）
  3. Gizmo をドラッグして引き出す
  4. 連続して E → ドラッグ → E → ... で階段状に伸ばせる

Edge モード — 辺の押し出し (fin)

  1. Edge モード（2）で辺を選択（境界辺がオススメ）
  2. E → 各辺に「fin」と呼ばれる 2 三角形のパッチが取り付けられる
  3. ループ状の辺を選んで extrude すると、共有頂点が dedup されて連続したスカートになる
内部辺 (内部の辺) を extrude すると 3 面が 1 辺を共有する non-manifold 状態になる （Blender も同じ仕様）。シルエットや fin geometry の追加に有用。

Vertex モード

未対応 — 三角メッシュでは「面に属さない単独辺」が表現できないため、エラーメッセージを表示して終了。

4.4 Inset (内側オフセット)

面の内側にスケールしたコピーを作る。Extrude と組み合わせて「ボス / ボタン」造形。キー: I

  1. Face モードで面を 1 つ以上選択
  2. I → 各面の内側に小さなコピーが作られ、cap が選択状態に
  3. 続けて E → ドラッグで「凹みボタン」や「凸ボス」

インセット量は右パネルの Inset Amount スライダーで調整（0 = なし、0.5 = 中心まで縮小）。

forge3d の Inset は Individual Faces モード（Blender の「個別面」）。 複数面を選んで I すると各面が独立してインセットされる（Region モードでは無い）。

4.5 Bevel (面取り)

辺を「面取り」する。直角コーナーを丸める時など。キー: Ctrl+B

  1. Edge モードで辺を選択（複数可）
  2. Ctrl+B → 選択辺が chamfer (面取り) クアッドに置換
  3. 幅は右パネルの Bevel Width スライダー (0〜0.49)

内部的には頂点ファンを分割し、暗黙のコーナー三角キャップで manifold を保つ高度な処理を行っている。

V2 制限: 1 頂点に同時に面取りできる辺は 1 本。共有頂点を持つ選択 （エッジループ全選択など）は、端点が重ならない交互のサブセットだけ面取りされ、 スキップ数がステータスに表示される。もう一度 Bevel を実行すると残りが面取りされる。 複雑なコーナー (Blender の "Branch" ケース) は V3 へ繰越。

4.6 Loop Cut (ループカット)

メッシュをぐるりと一周する辺のループを追加する。関節用ループや細部用ループの挿入に。キー: Ctrl+R

  1. Edge モードで 1 本の辺を選択（ループの seed として使われる）
  2. Ctrl+R → 自動でループを検出し、各辺の中点を結ぶ新しいループが挿入される
  3. 挿入後は Vertex モードに自動切替、新しい中点群が選択状態 → そのまま Gizmo で動かせる

ループ検出は「隣接 2 面の法線が平行か (コプラナリティ)」で四角形を推定して歩く。 立方体や円柱の側面など、quad-meshed なプリミティブで正しく動作する。

急角度のコーナー (法線の cos 値 < 0.7) ではループが切れて falls back に per-tri 分割。 通常は「切れて当然」な箇所なので望ましい挙動。

4.6.1 Edge Slide (エッジスライド)

選択した辺ループを、隣の辺 (レール) に沿って横滑りさせる。Loop Cut で入れたループの 「位置決め」に必須の操作。キー: G (Edge モード)

  1. Edge モードでスライドさせたい辺 (ループ) を選択
  2. 右パネルの Slide Amount スライダーで量を設定 (−0.95〜0.95、符号 = 方向)
  3. G または Edge Slide ボタン → 1 回押すごとに設定量だけ滑る (押すたび累積、各 1 Undo)

各頂点はループ接線に最も垂直な隣接辺をレールに選ぶので、三角化された quad メッシュでも対角線に引っ張られない。トポロジは変わらない (位置のみ)。 メッシュ境界やレールのない側では頂点はその場に留まる。

4.6.2 Vertex Slide (頂点スライド)

頂点 1 つを、隣の頂点と結ぶ辺に沿って滑らせる。Blender の Shift+V 相当。 キー: Shift+V (Vertex モード)

  1. Vertex モードで基準になる頂点 (アンカー) をクリック
  2. Ctrl+クリック で動かしたい頂点を追加選択 (計 2 頂点、辺で繋がっていること)
  3. Slide Amount スライダーで量を設定 — 正 = アンカーへ近づく、負 = アンカーから遠ざかる (辺の延長線上)
  4. Shift+V または Vertex Slide ボタン → 押すたび累積、各 1 Undo。選択は保持されるので連打で微調整できる

トポロジは変わらない (位置のみ)。辺で繋がっていない 2 頂点を選ぶとエラー。

4.7 Knife / Flip Diagonal

4.7.1 Knife (ナイフカット)

画面上にドラッグで線を引き、その線が横切るメッシュの辺をすべて切断して面を再分割する。 Blender の Knife (cut through ON) 相当。キー: K (どの選択モードでも可)

  1. K → ナイフモードに入る (ステータスバーに案内表示)
  2. ビューポート上で切りたい場所を横切るようにドラッグ (黄色の破線がプレビュー)
  3. 離すと切断実行 — 線が横切った辺すべてに新しい頂点が挿入され、面が分割される
  4. 実行後は Vertex モードに自動切替、切断線上の新頂点が選択状態 → そのまま Gizmo で動かせる

切断はカメラと切断線が成す平面で行われ、手前の面も奥の面も両方切れる (cut through)。切断範囲は引いた線分の画面上の長さに限定される (線を短く引けば部分カット)。 新しい頂点の UV / ボーンウェイトは切断された辺の両端から自動補間される (§4.9.4)。

オクルージョン判定はない (裏側の面も必ず切れる)。 手前だけ切りたい場合はビューを工夫して線の長さで範囲を絞る。1 カット = 1 Undo。

4.7.2 Flip Diagonal (対角線フリップ)

隣接 2 三角形の共有辺 (対角線) を、反対側の 2 頂点を結ぶ辺に張り替える。 キー: F

以前は「Knife」の名前で K キーに割り当てられていたが、本物の Knife (§4.7.1) 実装に伴い F = Flip へ移動した。
  1. Vertex モードで ちょうど 2 つの頂点を選択
  2. 2 頂点が「隣接 2 三角形の各 off-edge vertex」になっているとき有効
  3. F → 共有辺が消えて、2 頂点を直結する新しい辺に置換 (= edge flip)

三角化の整理に重宝。Bevel / Inset の後で対角線の向きを変えたい時など。

4.8 Delete

選択した要素を削除。キー: X / Delete / Backspace

  • Vertex モード: その頂点に触れる全ての面が消える
  • Edge モード: その辺の両側の面が消える
  • Face モード: 選択面が消える (頂点と辺は残る → Blender の "Only Faces" 相当)
削除で孤立した頂点は position buffer に残る (Blender の "Delete Loose" は別オペレータ)。 気になる場合は Object Mode の Mesh Tools → Weld Vertices で掃除できる。

4.8.1 Merge / Collapse (マージ / 辺収縮)

選択した要素を 1 点にまとめる。近すぎる頂点の整理、先端を閉じる、密度を落とす等に。 キー: M

  • Vertex モード: 選択頂点 (2 つ以上) を重心の 1 点にマージ (Blender の Merge At Center)
  • Edge モード: 各辺を中点に収縮 (Edge Collapse)。端点を共有する連続辺は まとめて 1 点になる

潰れた面 (頂点が重複した三角形) は自動で消え、未参照になった頂点はバッファから 除去される (シーム指定も追従)。実行後は Vertex モードに切り替わり、マージ先の頂点が選択される。

4.8.2 Bridge Loops (境界ループの橋渡し)

2 本の境界エッジループを四角形の帯 (三角形 ×2) で繋ぐ。腕と胴体の接続、 筒どうしの連結など。キー: Ctrl+E (Edge モード)

  1. Edge モードで 両方のループの辺をすべて選択 (Box Select B が便利)
  2. Ctrl+E または Bridge Loops ボタン
  3. 接続面が生成され、Face モードで選択状態になる
V1 制限: 両ループとも境界 (面が片側にしかない辺) で、 頂点数が同じであること。閉ループ同士 / 開パス同士のみ (混在不可)。 向かい合う筒の端など典型ケースでは巻き方向も自動で合う。頂点数が違う場合は 先に Loop Cut / Merge で数を揃える。

4.8.3 法線制御 (Shade Smooth / Flat / Auto Smooth)

面のシェーディングを「なめらか」か「カクカク (フラット)」か、あるいは折り目の角度で 自動切り分けするかを制御する。Object Mode の左パネル「Mesh Tools」から 実行する (メッシュ全体が対象、Edit Mode に入る必要はない)。

ボタン効果
Shade Smooth全頂点法線を平均化してなめらかに。球・キャラクター等の曲面向け。頂点数は変わらない
Shade Flat面ごとのフラットシェーディング。ハードエッジ部分で頂点が分割される。ローポリ・機械系向け
Auto Smooth ∠下の角度°入力より急な折り目だけハードエッジにし、 ゆるやかな面はなめらかに繋ぐ (Blender の Auto Smooth 相当)。30° 前後が定番

仕組み: 各面コーナーの法線を「そのコーナーの面と角度しきい値内で隣接する面」だけで平均し、 異なる法線になったコーナーで頂点を分割する。つまりハードエッジが必要な所だけ頂点が増える。 UV とスキンウェイトは分割後も元頂点から引き継がれる。全て Undo 可能。

モーフターゲットを持つメッシュには使えない (頂点数が変わると各モーフの 位置バッファが壊れるため)。表情メッシュは先にシェーディングを決めてからモーフを付けること。

4.9 Seam / UV Unwrap (シーム + UV 展開)

テクスチャを貼るための UV 座標を生成する。 オリジナルメッシュを Paint タブで塗るには UV が必須。

4.9.1 Seam (シーム) のマーキング

シーム = メッシュを「ハサミで切る」境界。UV 展開時にここで島が分かれる。

  1. Edit Mode の Edge mode (2) で辺を選択
  2. Shift+S または右パネルの Mark Seam ボタン
  3. 選択辺が赤色で表示される (シーム指定済み)
  4. もう一度押すと解除 (トグル)

4.9.2 Smart UV Project (展開実行)

  1. 必要に応じてシームを指定 (指定なしでも展開可能)
  2. U または右パネルの Smart UV Project ボタン
  3. メッシュが UV 展開され、各面が 0–1 の UV 空間内に配置される
  4. Edit Mode を抜けて Paint タブに切替 → テクスチャを塗ると正しく貼られる

4.9.3 アルゴリズム

Blender の Smart UV Project に相当。動作：

  1. 面を法線方向でクラスタリング (デフォルト 66° 以内で同クラスタ)
  2. シーム辺ではクラスタを分割
  3. 各クラスタを平均法線に垂直な平面に投影
  4. 島をシェルフ梱包 (rect-pack) で 0–1 UV 空間に配置 — 島のアスペクト比を保ったまま 高さ順に棚へ詰めるので、旧グリッド（全島を同一正方形セルに押し込む）より空間効率が高い。 全島で共通スケールを使うためテクセル密度が揃う

立方体ならシーム無しでも 6 つの島 (各面) に自動分割される。 球体や曲面で島を制御したいときは事前にシームを指定する。

制約:
  • UV 展開は頂点数を増やすが、クラスタ内 weld により最小限に抑えられる （同じ島内で共有される頂点は 1 つに統合、島の境界だけ複製 — Cube は 8 → 24 頂点。旧 split-per-face の 36 から改善）
  • スケルトン付きメッシュでも展開可能（ボーンウェイトは展開後の頂点へ自動で引き継がれる）。ただし Morph Target 付きは不可（エラーメッセージが出る — 頂点数変更にモーフが追従できないため、モーフ作成前に展開すること）
  • 展開結果のレイアウトは 🗺 UV Editor（§4.9.5）で調整できる

4.9.4 属性の引き継ぎ (UV / ボーンウェイト)

Extrude / Inset / Bevel / Loop Cut などトポロジを変える操作をしても、 既存頂点の UV とボーンウェイトはそのまま維持され、新しくできた頂点は 元サーフェス上の最近接点から自動補間で受け取る（バリセントリック補間、 ウェイトは上位 4 影響に正規化）。「UV 展開 → リグ → 形状のちょい直し」という 順序でも作業成果が壊れない。

4.9.5 UV Editor (2D UV ビュー)

展開済み UV レイアウトを 2D ビューで直接編集する全画面エディタ。 右パネル UV Unwrap セクションの 🗺 UV Editor ボタンで開く （Edit Mode 中のメッシュ、または Object Mode で選択中のメッシュが対象。UV 必須）。

  • Island モード (I)：島をクリックで選択し、ドラッグで変形。 変形は G=Move / R=Rotate / S=Scale で切替 （Rotate / Scale は島のバウンディングボックス中心がピボット）。 Flip U / Flip V ボタンで選択島を左右 / 上下反転
  • Vertex モード (V)：UV 頂点を個別にドラッグ移動。 Ctrl+クリック で複数選択に追加 / 解除。 Weld = 選択頂点を重心へ溶接（テクスチャの継ぎ目消し）、 Align U / Align V = 選択頂点を縦 / 横一直線に整列
  • 🌡 ストレッチ表示：面ごとのテクスチャ歪みをヒートマップ表示 （青=均一、緑〜赤=引き伸ばし / せん断 / テクセル密度のずれ）。編集中もライブ更新される
  • ⌗ Snap：移動をグリッド (1/4〜1/32、薄いガイド線表示) に吸着。 Vertex モードでは他の UV 頂点へのマグネットが優先され、近づけるだけで ぴったり重なる (→ そのまま Weld で溶接)。島は bbox 左下角が吸着基準。 Snap 中は回転 15° / 拡縮 0.1 刻みに量子化
  • ナビゲーション：ホイール=ズーム、中 / 右ドラッグ=パン
  • 編集内容は 3D ビューポートのテクスチャにライブ反映。 Apply & Close で確定（1 Undo）、Cancel / Esc で破棄
背景の市松模様は 0–1 UV 空間。島をこの枠内に収めるとテクスチャが正しく貼られる。 ストレッチ表示が赤い島は、3D 上の形と UV 上の形が乖離している — シームを増やして再展開するか、 Scale で他の島とテクセル密度を揃えると改善する。

5. Sculpt (スカルプト)

有機的な凹凸を「塗る」感覚で作る。筋肉、しわ、形状調整に。キー: D

5.1 ブラシの種類

ブラシ効果
Push頂点を法線方向に押し出す (凸)
Pull頂点を法線方向に引っ込める (凹)
Smooth近傍頂点の平均位置に滑らかに
Flatten平面に近づける

5.2 パラメータ

  • Radius: ブラシ半径 (ワールド単位)
  • Strength: 1 ストロークあたりの変位量
  • Falloff: 中心から外側への減衰指数
高ポリゴンメッシュには空間グリッド最適化が効いて、数万頂点でも対話的に動く。

5.3 Symmetry (対称)

パネルの X / Y / Z チェックでブラシをオブジェクトローカルの各軸平面に ミラーする。複数軸を同時に有効化でき (X+Y で 4 箇所、X+Y+Z で 8 箇所に同時適用)、 左右対称のキャラクターを片側だけ彫れば反対側も自動で同じ形になる。対称平面上の ストロークは二重適用されない。Mask ブラシにも対称が効く。

5.4 Dyntopo (動的トポロジ)

適応的細分化 をオンにすると、ストローク中にブラシ半径内で Detail より長いエッジが自動的に中点分割される。元メッシュのポリゴン数が 少なくても、彫り込んだ部分にだけディテール用の頂点が増えていく (隣接三角形と中点を共有するので穴が空かない)。

  • Detail: 分割を始めるエッジ長 (ワールド単位)。小さいほど細かくなる。
  • 細分化は彫る方向 (subdivide) のみ。短いエッジの間引き (collapse) は将来対応。
  • UV とマスクは細分化後も補間されて引き継がれる。
  • スキン (ボーンウェイト) 付きメッシュでは Dyntopo は無効 (ジオメトリ再構築でウェイトが失われるため)。リギング前に彫ること。
Undo はストローク単位で、Dyntopo によるトポロジ変化 (頂点数・面数の増加) も 含めて 1 回で元に戻せる。Mask の塗り変えも個別に Undo 可能。

5.5 マスクのクリア

▦ Clear Mask ボタンで選択メッシュの保護をすべて解除する。 マスクが塗られた頂点はビューポート上で暗く表示される (頂点カラーによる可視化)。

6. Paint (テクスチャペイント)

UV 座標を持つメッシュの表面に色を塗る。キー: P

  • 動的テクスチャに直接描画。Resolution (512 / 1024 / 2048) は 新しく作られるペイントテクスチャに適用される (既存はそのまま。ブラシサイズは 解像度に合わせて自動スケール)
  • カラーピッカーで色選択、ブラシサイズ・不透明度・消しゴムモードあり
  • Hardness: ブラシ縁の硬さ。1 = くっきりした円、0 = エアブラシ (中心から滑らかにぼける radial falloff)。頬の赤みや陰影のグラデーションに
  • ストロークは連続: ポインタイベント間をブラシ間隔で補間するので、 速くドラッグしても点線にならず 1 本の線になる。UV シームをまたいだ瞬間は 自動検出して補間を切る (別の島に線が飛ばない)
  • UV 座標がないメッシュには使えない (警告メッセージ表示)
  • テクスチャインポートで読み込んだアルベドの上にペイント可能
  • Undo はストローク単位 (1 ストローク = 1 Undo)
プリミティブは生成時に UV が自動付与される。インポートメッシュは元の UV を尊重。 2048 使用時は Undo スナップショットが縮小保存されるため、Undo 後にわずかなぼけが出ることがある。

6.1 ペイントレイヤー

Paint タブの Layers セクション。Photoshop 的なレイヤースタックで、 最下段の Base (不透明、削除不可) の上に透明レイヤーを重ねられる。 ストロークはアクティブレイヤー (ハイライト行) にだけ乗る。

  • + Add Layer: 透明レイヤーを最上段に追加してアクティブ化 (Undo 可)
  • 行クリック = アクティブ切替 / チェック = 表示・非表示 / ✕ = 削除 (Undo 可)
  • ブレンドモード: normal / multiply (乗算 — 陰影・汚し) / screen (スクリーン — ハイライト) / overlay (オーバーレイ — コントラスト)。 Opacity スライダーと組み合わせて非破壊で重ねる
  • 消しゴムはレイヤーで挙動が変わる: Base ではベース色で塗り戻し、 上位レイヤーでは透明に消して下のレイヤーを見せる
  • Clear Paint はアクティブレイヤーだけをクリア
  • ストローク Undo / Clear Undo もレイヤー単位で正しく戻る
V1 制限: レイヤー構成はセッション内のみ。 GLB / .forge3d には合成結果のテクスチャが保存される (開き直すと 1 枚に統合された 状態から再開)。表示/blend/opacity の変更は Undo 履歴に乗らない (いつでも触り直せる live 値)。

6.2 チャンネルペイント (Roughness / Metallic)

Paint Brush の Channel セレクタで、色 (Albedo) 以外に Roughness と Metallic を直接ペイントできる。 「目だけツヤツヤ」「ベルトの金具だけ金属」のような質感の塗り分けに。

  • 値の指定はカラーピッカーの明るさ (Rec.709 輝度): 白 = 1.0 (完全にザラザラ / 完全に金属)、黒 = 0、グレー = 中間。ステータスバーにも案内が出る
  • 初回ストロークで現在のマテリアルの Roughness / Metallic 値を初期値として テクスチャ化し、マテリアルをテクスチャ駆動モードに切替える (glTF 標準の metallic-roughness パック: G = roughness / B = metallic)
  • 消しゴムはそのチャンネルの初期値に塗り戻す
  • ブラシの Size / Opacity / Hardness、連続ストローク、1 ストローク = 1 Undo は Albedo と同じ
  • GLB エクスポートでは metallicRoughnessTexture としてそのまま出力される
チャンネルペイントはレイヤー非対応 (フラット 1 枚)。 レイヤーは Albedo 専用。Metallic スロットに画像をインポートするとチャンネルペイントは破棄される。

7. Bone / Rigging (ボーン / リギング)

キャラクターを動かすためのスケルトンを組む。キー: B

7.1 ボーン作成

  1. Bone ツール (B) に入る
  2. ビューポートをクリックでその位置にボーンを追加 (球状のビジュアル表示)
  3. 既存ボーンをクリックして親選択 → 次のクリックで子ボーン

7.2 編集モードとポーズモード

モードGizmo用途

右パネルから切替。Edit でレイアウト → Pose でアニメ用のポーズを作る流れ。 Pose の回転ドラッグは現在のポーズに累積する（ドラッグを重ねるほど回っていく。 戻したいときは Undo か、逆方向へドラッグ）。

7.2.1 Pose 回転のローカル軸 (Local 軸回転)

Mode セクションの Local 軸回転（デフォルト ON）で、Pose の回転ギズモの リングをボーン自身の軸に整列させる。ボーンの +Y 軸を「自分の位置 → 最初の子」方向 （子が無い末端ボーンは「親 → 自分」方向）に取り、そのまわりの捻りは選択ボーンの Roll°（§7.3.1）で調整できる。Blender のギズモ方向 Normal 相当。

  • ON（Local）: リングがボーンに沿う — 「ボーン軸まわりに捻る」「関節を曲げる」が直感的
  • OFF（World）: 従来どおりワールド X/Y/Z 軸のリング

設定はブラウザに永続化される（§2.1.1 と同じ prefs 機構）。どちらのモードでも ベイクの数学は共通で、回転はドラッグ量ぶんだけ正しく累積する。

7.2.2 ポーズのコピー / ペースト / ミラーペースト

Mode セクションの 3 ボタンで、ボーン単位のポーズ（ローカル回転＋位置）を移送する。すべて Undo 可。 Auto-Key が ON なら、ペーストしたポーズはギズモ操作と同様に現在フレームへ自動キーされる。

ボタン動作

📥 Pasteコピーしたポーズを選択中のボーンへ貼り付け
🪞 Mirrorコピー元の対側ボーン（_L↔_R 等、§7.3 の命名規約で解決）へ X ミラーしたポーズを貼り付け。Blender の Paste Pose Flipped 相当。 左腕をポーズ → Copy → Mirror で右腕が対称ポーズになる

7.3 ボーンの操作

  • ボーンクリック → 選択 → Gizmo で動かす
  • 右パネルの Bone リストから名前変更 / 削除 / IK 設定
  • ⇄ Mirror Bone (X): 選択ボーンとその子孫を X=0 平面で左右反転コピー。 名前の _L/_R（および Left/Right、.L/.R 等）を自動でスワップし、 IK 設定もミラーした状態で複製する。左半身だけリギングして右半身を一発生成する用途。 反転集合の外にある親（背骨など）には共有のまま接続される。1 操作 = 1 Undo。
  • Shift+H: ボーン表示トグル
命名規約は arm_L_upper / hand_R のように区切り（_ . - 空白）で 囲まれた L/R、単語 Left/Right、および camelCase（armL↔armR、 LArm↔RArm）を side トークンとして認識する。Lever/Ball のように 単語の一部の L/R は誤検出しない。side トークンが無い名前（spine 等）は _mirror が付く。chiikawa-reign 連携用ボーン名（§13）に合わせると後工程が楽。

7.3.1 ボーン Roll（レスト方向の捻り）

Selected Bone パネルの Roll° は、ボーン軸（+Y = 子ボーン方向）まわりの レスト方向の捻り（度）。Blender のボーン Roll に相当し、Local 軸回転ギズモの リングの向きを制御する。肘・膝など「曲げ軸」を一定方向に揃えたいときに、 ギズモの X/Z リングが関節の曲げ方向と一致するよう調整する。

  • メッシュは変形しない（ギズモの整列にだけ効く軽量なメタデータ）
  • 編集は 1 ジェスチャ = 1 Undo
  • .forge3d プロジェクト（§13.4）にボーン名キーで保存・復元される（GLB 単体では保持されない）

7.4 メッシュへのアサイン

右パネルの「Assign Skeleton」でメッシュにスケルトンを割り当て → Weight Paint へ進む。

7.5 ボーン表示の調整 (Display)

Bone タブの「Display」セクションで見た目を調整できる：

  • Bone Size（0.25〜4.0、デフォルト 1.0）— ボーンの球ビジュアルの大きさ。 大きなメッシュには 2.0〜3.0、細かい作業時は 0.5 など。新規ボーンも現在の設定を継承する。
  • X-Ray（デフォルト ON）— メッシュ越しにボーンを表示。 キャラクター内側に埋まったボーン（脊椎、骨盤等）を見ながらリギングできる。 Blender の Armature "In Front" 相当。 OFF にすると通常のデプステストでメッシュに隠れる。
  • Shift+H: ボーン全体の表示/非表示トグル（モデリング中など邪魔な時に）
X-Ray の実装：bone visual と hierarchy line を renderingGroupId = 1 に置き、material の disableDepthWrite = true + depthFunction = ALWAYS で深度テストを無効化している。

7.6 IK (インバースキネマティクス)

チェーンの先端 (手・足など) を目標位置へ向け、途中の関節を自動で曲げる。 ボーン長を保ったまま解く FABRIK ソルバを採用。Bone タブの「IK Constraint」セクションで設定する。

  1. チェーンの先端ボーンを選択（例：手首）
  2. Enable IK をオン
  3. Chain Length（1〜5）で先端から数えて何本の関節を曲げるか指定（例：腕なら 2＝肘＋肩）
  4. Snap to Bone で目標を現在の先端位置に置いてから、Target X/Y/Z で目標を動かす
  5. Solve IK Now で一度だけ解く（Undo 可能）。または Enable IK のままにすると毎フレーム自動で追従する
項目意味
Enable IK毎フレームの自動追従をオン。オフでも「Solve IK Now」は使える
Chain Length曲げる関節数（先端含む、最低 2 本）。ルート側は固定される
Target X/Y/Z先端が到達しようとするワールド座標。マゼンタの球マーカーで可視化
Pole (bend dir)肘・膝がどちら向きに曲がるかを制御するポール（極ベクトル）。 ON にすると Pole X/Y/Z 方向へ中間関節が向く。Snap to Bend で現在の曲がり方向に自動配置
Max Bend °各関節の最大曲げ角（度）。0＝無制限。きつくすると鋭い折れ曲がり／過伸展を防ぐ。 制限が強いほど到達範囲は狭まる（先端が目標に届かないことがある）
Solve IK Now1 回だけ解いて履歴に積む（ボタン操作向け）
ポールの仕組み：FABRIK で解いた後、ルート→先端の軸まわりに中間関節を剛体回転させて 曲がりをポール方向へ向ける。軸はルートと先端を通るので、先端位置とボーン長は厳密に保たれる （曲がり方向だけが変わる）。3 関節未満（中間関節なし）のチェーンでは無効。 腕・脚で「肘が裏返る／膝が逆に曲がる」を防ぐのに使う。
IK はボーン行列を直接書き換えるため、スキニング済みメッシュが実際に変形する。 目標がチェーンの届く範囲を超えると、チェーンは目標方向へまっすぐ伸びた「最も近い姿勢」になる (ステータスに out of reach と表示)。アニメ再生時、IK 有効ボーンは各フレームで解かれる。

7.7 ボーンコンストレイント (Limit Rotation / Aim)

ボーンの回転に「守らせたいルール」を宣言する。Bone タブの「Bone Constraints」セクションで設定し、 IK の後に毎フレーム自動適用される（手付けポーズにも IK の解にも効く）。Undo 履歴には積まれない —— ポーズ操作ではなくリグの設定として扱われる。

項目意味
Aim (注視)ボーンの +Y 軸（Roll 反映、§7.3.1）をワールド座標のターゲットへ向け続ける。 Blender の Damped Track 相当。目・頭の注視、砲塔の追尾などに。 Snap Ahead でターゲットを現在のボーン方向の先へ置いてから動かすと、有効化の瞬間にポーズが跳ばない
Limit Rotationローカル回転を軸ごとに min/max（度）へクランプ。Blender の Limit Rotation 相当。 軸ごとのチェックで制限する軸だけ選ぶ（チェックなしの軸は自由）。肘・膝が逆に曲がらないようにしたり、 首の可動域を制限したりするのに使う
適用順は IK → Aim → Limit Rotation（ボーンは親→子の順に評価）。 Aim が回転を「作り」、Limit がその結果を「削る」。IK 中間関節に Limit Rotation を付ければ 関節可動域として働く（IK 側の Max Bend ° はチェーン全体の曲げ制限、こちらは軸別の細かい制限）。 🪞 Mirror Bone は Aim ターゲットを反転座標へ、Limit の範囲を反射則（Y/Z 軸の [min,max] → [-max,-min]）で、 Roll を符号反転してコピーする。

8. Weight Paint (ウェイトペイント)

各頂点がどのボーンにどれだけ影響されるかを塗る。キー: W

8.1 Auto Weights (自動ウェイト)

手塗りの前に、まず ⚡ Auto Weights で全頂点を最寄りのボーンへ一括バインドするのが基本ワークフロー。 各ボーンを「親関節→自関節」の線分とみなし、頂点から各線分への距離の逆数（距離が近いボーンほど強い）で ウェイトを決め、最大 4 ボーンに正規化する。

  1. スケルトンをメッシュにアサイン (§7.4) 済みであること
  2. 対象メッシュを選択して ⚡ Auto Weights を押す
  3. 結果を Show/Hide Overlay で確認し、気になる箇所だけ下のブラシで手直しする

Geodesic (surface)（デフォルト ON）：距離をメッシュ表面に沿って測る。 直線距離だと、空間的には近いが面としては繋がっていない部位（例：腕の先端が胴体に近づいたとき）に ウェイトが「漏れる」。Geodesic は頂点接続グラフ上の最短距離（Dijkstra）でバインドするので、 腕のウェイトが胴体に飛ばない。インデックス（三角形）が無いメッシュでは自動で直線距離にフォールバックする。

Auto Weights は既存のウェイトを上書きする（Undo 可能）。 脇・股など近接ボーンが混ざりやすい部位は、Geodesic でかなり改善するが、それでも気になれば 自動バインド後に Smooth ブラシで均すと綺麗になる。シームで頂点が分割されたメッシュも 同一座標の頂点を溶接してグラフを繋ぐので問題ない。

8.2 手塗り

  1. スケルトンが割り当てられたメッシュを選択
  2. ボーンを 1 つ選択 (Weight Paint 中もボーンビジュアルが表示)
  3. メッシュ上をドラッグでウェイトを塗る
モード動作
Addウェイトを加算 (デフォルト)
Subtractウェイトを減算
Smooth近傍と平均化

Ctrl/Shift キーで一時的にモード切替可能。1 頂点あたり最大 4 ボーンの影響を保持 (Babylon の標準仕様)。

塗ったウェイトはグレースケールオーバーレイで可視化される (明るい = そのボーンに強く影響)。

9. Animation (アニメーション)

ボーンを動かして時間軸上にキーフレームを置く。キー: A

9.1 クリップ管理

  • 右パネルの Anim タブで「New Clip」 → 名前と Frame Rate / Max Frames を設定
  • 複数クリップ作成可。Active ドロップダウンでいつでも編集対象のクリップを切替できる (idle / walk / attack を並行して作り分ける)。切替時はタイムラインのスライダーも選択クリップの設定に同期する

9.2 キーフレームの作成

  1. フレーム位置を Timeline で指定
  2. ボーンを Pose モードで動かす
  3. 「Key Bone」(個別) または「Key All」(全ボーン) でキャプチャ (どちらも Undo 可能)

9.2.1 Auto-Key (自動キー)

Record セクションの 🔑 Auto-Key (デフォルト ON) が有効な間は、 Pose モードでボーンの回転ドラッグを終えるたびに、そのボーンのキーが現在フレームへ 自動記録される (1 操作 = 1 Undo)。OFF の場合、キー未記録のままタイムラインを スクラブするとポーズはクリップの値で上書きされる — このとき警告が表示されるので、 残したいポーズは Record してからスクラブすること。Auto-Key は Morph スライダー (§9.2.2) にも効く。

9.2.2 モーフ (表情) のキーフレーム

ボーンと並行して、Morph Target の influence もクリップにキーを打てる。 まばたき・口パク・表情変化のアニメーションはこれで作る。

  1. Morph タブでターゲットを作成し (§12.1)、フレーム位置を指定
  2. Morph スライダーで表情を作る (Auto-Key ON ならスライダーを離すと自動キー)
  3. または ⏺ Record Morphs で全ターゲットの現在値を一括キー
  4. ✕ Delete Morph Keys で現在フレームのモーフキーを削除 (どちらも Undo 可)
  • 補間はボーンと同じ規則 (キー間はイージング、両端はクランプ)。キーごとの easing 指定に対応
  • ドープシートにティール色のモーフレーンとして表示される (ボーン行の下)
  • GLB 出力対応: クリップごとの AnimationGroup に influence アニメが合成され、 glTF 標準の morph weights アニメーションとして出力される。スケルトンなしのメッシュ (表情だけのモデル) でもクリップが出力される
  • Graph Editor でカーブ編集可能: ツールバーの Morphs トグルで 全モーフトラックの influence カーブ (0–1 軸、ティール系色分け) を表示。キー (●) を ドラッグで時間と値を同時編集 (値は 0–1 にクランプ、1 ドラッグ = 1 Undo、 移動先に既存キーがあれば時間は据え置き)。背景クリックでスクラブ

9.3 編集

  • Dopesheet: キーフレーム一覧を時間軸上で表示・編集 (§9.3.1)
  • Graph Editor: 個別チャンネルのカーブ表示、キーの移動 (§9.3.2)、Bezier タンジェント編集
  • Easing: Linear / Ease In/Out/InOut の組み合わせを各キーに設定可能
  • キーフレームのコピー / ペースト / 削除

9.3.1 Dopesheet のキー編集 (リタイミング)

ドープシートのキー (◆) はドラッグで時間移動できる。

  • クリック: そのキーへスクラブ + そのキーを単独選択 (行のボーンも選択される)
  • Shift+クリック: 複数選択のトグル。行をまたいだ選択も可 (腕と脚のキーをまとめて動かす等)。モーフレーンのキーも選択できる
  • ドラッグ: 選択中の全キーを剛体移動 (相対間隔を保ったまま)。ドラッグ中は 移動先に半透明ゴーストが表示され、ステータス行に Δフレームが出る。全キーが 0〜Max Frames に 収まるよう自動クランプ
  • ドロップ: 1 回のドラッグ = 1 Undo (Retime Keys)。移動先に別の (未選択) キーがあった場合は上書きされる (Blender 等と同じ)
  • 空白部分のクリックは従来通りスクラブ + キー選択解除
  • Shift+ドラッグ (空白から): ボックス選択。黄色い矩形内の キーがすべて選択に追加される (行・モーフレーンまたぎ可)。まっさらから 選び直したいときは先に空白をクリックして選択解除

9.3.2 Graph Editor のキー移動 (複数選択対応)

カーブ上のキー (●) をドラッグで時間と値を同時に編集できる (1 ドラッグ = 1 Undo)。 タンジェントハンドル (○) が優先でヒットし、次にキー、どちらでもなければスクラブ。

  • Shift+クリック: キー点を複数選択にトグル (チャンネルまたぎ可)。 選択中の点は白枠付きの大きい点で表示される
  • 選択中の点をドラッグ: 選択した全キーを剛体移動 — 時間 Δ はキーフレーム単位で共通、値 Δ は選択した各チャンネルに同量加算。 クリップ範囲に収まるよう自動クランプ
  • 移動先フレームに未選択キーがある場合、時間は動かず値だけ変わる (上書きしたいときは ドープシート §9.3.1 で)
  • 背景クリックで複数選択を解除 + スクラブ

9.4 再生

Timeline 下の Play ボタンで再生プレビュー。Loop Mode を Cycle / Constant で切替。

9.4.1 オニオンスキン (👻 Onion Skin)

Timeline セクションの 👻 Onion Skin を ON にすると、現在フレームの ±Offset フレームのポーズがゴースト骨格 (ワイヤ) として表示される — 前 = 緑、次 = 赤。ポーズの来し方と行き先が見えるので、動きの緩急やアークの 確認がスクラブなしでできる。Offset (1〜20f) はスライダーで調整。ゴーストはクリップの キーフレームデータから純粋に計算されるので、実際のボーン / メッシュには影響しない。 設定はブラウザに保存され次回起動時も維持される。

12.1 Morph Target (Shape Key)

顔の表情など、複数の形状をブレンドする仕組み。右パネルの Morph タブ。

  • ベース形状から複数のターゲット形状を作成
  • 各ターゲットの influence (0-1) で混合
  • GLB エクスポートで morph target として保存
  • influence はアニメーション可能 — クリップにモーフキーを打てる (§9.2.2)。 Auto-Key ON ならスライダーを離した瞬間に現在フレームへ自動キー

12.1.1 Shape Key Driver (ドライバ)

ボーンの姿勢チャンネルでモーフの influence を自動駆動する。Blender の 「シェイプキードライバ」相当 — 肘を曲げると筋肉が膨らむ補正シェイプ、首をかしげると 頬がつぶれる、といったリグ連動の自動表情 / 補正に使う。

  1. スケルトンがある状態で Morph タブを開くと、各ターゲットの下に ⚙ ドライバ行が出る
  2. ⚙ チェックで有効化 → ボーン / チャンネル (Rot X〜Pos Z) / Min / Max を設定
  3. ボーンを動かすと influence が自動で追従: チャンネル値が Min で 0、Max で 1 (線形、範囲外はクランプ)。 Min > Max にすると逆方向マッピング
  • 評価は毎フレーム、IK / ボーンコンストレイントの後に行われる (確定したポーズを読む)
  • ドライバ有効中はスライダーとモーフキーよりドライバが優先 (スライダーは無効表示になる。Blender と同じ「駆動値は直接編集不可」の思想)
  • ドライバは .forge3d プロジェクトに保存される (§13.4、メッシュ / ボーン名で対応付け)。 GLB には乗らない (glTF にドライバ概念がないため)
回転チャンネルの単位はラジアン。「肘 90° で最大」なら Min 0 / Max 1.57。

12.4 Layer / 親子付け (Parenting)

  • Layer（コレクション）: メッシュをグループ分けして表示/非表示を一括制御。 行の + ボタンでサブコレクションを作成でき、階層でネストする （Blender のコレクション相当）。親を非表示にすると配下のレイヤーのメッシュもまとめて隠れる （継承された非表示は行が薄く表示される）。レイヤー削除時は子レイヤーが親へ付け替わる。 階層構成は .forge3d プロジェクトにも保存される

12.5 Procedural Material (手続きマテリアル)

Material タブの「Procedural」セクションで、ノイズ・チェッカー・グラデーションといった 手続きパターンを CPU で評価してテクスチャに焼き付け、PBR の Albedo に適用する。 焼いたテクスチャは通常のテクスチャなので GLB エクスポートにそのまま乗る（§13）。

  1. メッシュを選択（マテリアルが無ければ自動で PBR が作られる）
  2. プリセット Noise / Voronoi / Brick / Checker / Gradient を選ぶ
  3. Scale / Contrast / Color A / Color B（Gradient は Vertical）を調整
  4. 必要なら Roughness from pattern（ラフネス連動）/ Bump from pattern（法線マップ生成、Bump Strength で凹凸の強さ）を ON
  5. ⚡ Bake to Albedo で焼き付け。以降パラメータを動かすと即座に再ベイクされる（プレビュー）
  6. Clear Procedural で解除
プリセットパラメータ
NoiseScale（粒の細かさ）/ Contrast / Color A↔B（暗→明のランプ）
VoronoiScale（セルの数）/ Contrast / Color A↔B。石・細胞・ひび風
BrickScale（段数）/ Contrast（目地の太さ）/ Color A（目地）↔B（レンガ）
CheckerScale（マス目の数）/ Color A・B
GradientVertical（U→V 方向切替）/ Color A→B

Roughness from pattern：ON にすると、同じパターン値で PBR のラフネスも駆動し、 glTF 標準の metallic-roughness テクスチャ（R=AO / G=roughness / B=metallic）を一緒に焼く。 レンガの目地だけツヤを変える、ノイズで使い込んだ表面を作る等、質感が一段リアルになる。これも GLB に乗る。

Bump from pattern：パターンを高さ場とみなして接空間ノーマルマップ（R=x/G=y/B=z）を 中心差分で生成し、PBR の bump（法線）に適用する。Bump Strength で凹凸の強さを調整。 レンガ・ボロノイ・ノイズが立体的に見え、これも glTF の normalTexture としてそのまま GLB に乗る。

ノードグラフ編集（Edit Node Graph）

プリセットを超えて任意のグラフを組みたいときは 🕸 Edit Node Graph で全画面のノードエディタを開く。 内部は「UV → ノイズ/ボロノイ/レンガ/チェッカー/グラデ → Math/カラーランプ/ミックス → 出力」のノードグラフで、 プリセットもこのグラフを生成しているだけ。Math ノード（Add/Sub/Mul/Min/Max/Average）で パターン同士を重ね（例：ノイズ×ボロノイ）、Mix ノードはモード（Mix/Add/Multiply/Screen）で色を合成できる。

  • ノード追加：上部パレットの「+ Noise」等をクリック
  • 移動：ノードのヘッダをドラッグ
  • 結線：出力ポート（右の丸）から入力ポート（左の丸）へドラッグ。型不一致（float↔color）は自動変換、 循環は自動で拒否
  • 切断：接続済み入力ポート（白枠）をクリック
  • 削除：ノードヘッダの ✕（出力ノードは削除不可）
  • パラメータ：各ノード内のスライダ/カラーで調整。変更は即プレビュー（メッシュに反映）
  • Apply & Close で確定（Undo 1 件）、Cancel で編集前に戻す
  • Output ノードの albedo / roughness / metallic / height 入力にそれぞれ色・値を繋ぐと、対応する PBR チャンネル（および metallic-roughness / normal マップ）が焼かれる
  • ⬇ Export / 📂 Import：組んだグラフを JSON で保存・読込。カスタムマテリアルを再利用できる （不正な JSON は検証で弾かれる）
評価・ベイク（albedo / metallic-roughness / normal）とグラフ操作（追加・削除・結線・循環検出）は 純関数として実装されテスト済み。キャンバス描画のみ DOM 層。Bake / Clear / Edit は Undo 可能。解像度はデフォルト 256px。

12.6 AO Bake (アンビエントオクルージョンベイク)

ジオメトリのセルフオクルージョン（へこみ・すき間・接地部が環境光を受けにくい度合い）を UV レイアウトに沿ったテクスチャに焼き付け、PBR の AO スロットに適用する。 Material タブのテクスチャスロット欄の AO Bake 行（解像度 256 / 512px 選択 + 🔥 Bake）。

  1. UV 必須 — 先に Smart UV Project (§4.9) を実行しておく
  2. メッシュを選択 → Material タブ → 🔥 Bake
  3. 数秒でベイク完了。しわ・耳の付け根・脚の間などが自然に暗くなる
  • 各テクセルから半球状にレイを飛ばして遮蔽率を計算（コサイン加重 16 サンプル、 到達距離はメッシュサイズの 25%）。均一グリッドで加速しているので 512px でも実用時間
  • サンプリングは決定的 — 同じメッシュなら何度焼いても同じ結果
  • UV 島の境界は自動で外側に膨張させるので、シームに黒い筋が出ない
  • GLB エクスポートでは glTF 標準の occlusionTexture として出力される (§13)
  • 1 ベイク = 1 Undo。AO スロットの ✕ でいつでも解除できる

12.7 Normal Bake (ハイ→ロー法線ベイク)

retopo ワークフローの要：スカルプトで作り込んだ高密度メッシュ（ハイポリ）の 表面ディテールを、軽いローポリメッシュのノーマルマップに転写する。 Material タブの Nrm Bake 行 — ハイポリのメッシュを選んで 🔥 Bake。

  1. ローポリ側（ベイク先）を選択。UV 必須 (§4.9)
  2. Nrm Bake 行のドロップダウンでハイポリ（ソース）メッシュを選ぶ
  3. 🔥 Bake → ノーマルマップが Normal スロットに入り、ローポリが彫り込んだように見える
  • ローポリ表面の各テクセルから法線方向にケージレイを飛ばし （オフセットはメッシュサイズの 5%）、最近接のハイポリ面の平滑法線をサンプル → ローポリの接空間（UV から計算したタンジェント基底）に変換して RGB 符号化 （glTF / OpenGL 規約、512px）
  • ワールド空間でベイクするので、2 つのメッシュは見た目が重なっていれば OK （トランスフォームの違いは吸収される）
  • レイが届かなかったテクセルはフラット法線 (128,128,255) にフォールバック。 ヒット率が 50% を切ると警告 — ハイポリとローポリが重なっているか確認
  • GLB エクスポートでは glTF 標準の normalTexture として出力される (§13)
  • 1 ベイク = 1 Undo。Normal スロットの ✕ で解除
典型的な流れ：ローポリを複製 → 複製に Subdivision + Sculpt でディテールを彫る → 元のローポリを選択して複製をソースに Normal Bake → 複製を削除（または非表示レイヤーへ）。

  • 中身 = シーン全体の GLB (メッシュ / マテリアル / スケルトン / アニメ / モーフ) + GLB に乗らない編集データのサイドカー JSON
  • サイドカーが保持するもの: 手続きマテリアルのノードグラフ (再編集可能なまま) / Sculpt マスク / レイヤー構成と割り当て / ボーン Roll (§7.3.1、ボーン名キー) / ボーンコンストレイント (§7.7、Limit Rotation / Aim、ボーン名キー) / Shape Key Driver (§12.1.1、メッシュ / ボーン名で対応付け)
  • 開くと通常のモデル読込と同じ流れでシーンに追加され、グラフは自動再ベイク、 マスクとレイヤーも復元される (メッシュ名で対応付け)
  • 非可搬なもの (現状): モディファイアスタック (エクスポート時点の評価結果が形状として保存される)

15. 用語解説

用語意味
Mesh頂点 (vertex) と三角形 (face) で表現された 3D 形状。Forge3D 内部はすべて三角メッシュ。
Vertex / 頂点3D 空間の点。位置 (x, y, z) を持つ。
Edge / 辺2 つの頂点を結ぶ線分。
Face / 面3 つの頂点で囲まれた三角形。法線 (どちらが外側か) を持つ。
Normal / 法線面 (または頂点) の「外向き方向」を示すベクトル。光の当たり方を計算するのに使う。
UV3D 表面を 2D に展開した座標 (u, v)。テクスチャの貼り方を決める。
Triangulation / 三角形化四角形以上の多角形を三角形に分割すること。Forge3D の内部表現は常に三角化済み。
Manifold「閉じていて穴がない」「各辺がちょうど 2 面に共有される」状態。3D プリントや物理計算で重要。
Boundary edge / 境界辺1 つの面にしか属していない辺。Manifold な閉メッシュには存在しない。開いたメッシュ (Plane や open shell) の縁。
Non-manifold3 つ以上の面が 1 辺を共有している状態など、CG として「不正」または特殊な状態。意図的に作ることもある (fin geometry など)。
Half-Edge各辺を「向きのある半辺」として表現するデータ構造。Edit Mode の内部実装で使用 (各 face が 3 つの half-edge を持ち、twin で隣接 face と繋がる)。
Implicit Quad三角メッシュで、隣接する 2 つの三角形を「論理的な四角形」として扱う考え方。Loop Cut の検出に使う。
Diagonal / 対角線四角形を 2 つの三角形に分けるときの分割線。三角化の方向を決める。
Gizmoビューポート上のドラッグ可能な操作ハンドル (3 軸の矢印 / 円 / 角キューブ)。位置・回転・拡縮に使う。
Centroid / 重心選択頂点の平均位置。Edit Mode の Gizmo はここに配置される。
Skin / Skinningメッシュをスケルトンに「皮膚」として貼り付けること。各頂点が複数ボーンの影響を受ける (weight)。
Bone Weight頂点ごとの「このボーンに何 % 影響される」の重み (0-1)。合計が 1 になるよう正規化される。
Rest Poseキャラクターの「初期姿勢」。アニメーション前の基準ポーズ。
Roll (ボーン)ボーン軸 (+Y = 子ボーン方向) まわりのレスト方向の捻り。Pose の Local 軸ギズモのリングの向きを決める。Blender のボーン Roll 相当。§7.3.1 参照。
Paste Pose Flippedコピーしたポーズを左右反転して対側ボーン (_L↔_R) へ貼り付ける操作。Forge3D では Bone タブの 🪞 Mirror ボタン。§7.2.2 参照。
IK (Inverse Kinematics)「先端をここに置きたい」から「関節をどう曲げる？」を逆算する仕組み。手足の制御に使う。
Bone Constraint / ボーンコンストレイントボーンの回転に毎フレーム適用されるルール。Forge3D V1 は Limit Rotation (軸別の回転クランプ) と Aim (注視追従) の 2 種。§7.7 参照。
Aim / Damped Trackボーンの軸をターゲットへ向け続けるコンストレイント。Blender の Damped Track 相当。視線・砲塔などに。§7.7 参照。
Keyframe時間軸上の「ここでこの値」を記録する点。間は補間される。
Easingキーフレーム間の補間カーブ。Linear (一定) / Ease In/Out など。
Retiming / リタイミングキーフレームの時間位置を変えてアニメの緩急を調整すること。Forge3D ではドープシートのキーをドラッグ。§9.3.1 参照。
Onion Skin / オニオンスキン前後フレームのポーズを半透明ゴーストで重ね表示するアニメ確認手法。前=緑 / 次=赤。§9.4.1 参照。
Bezier handleキーフレームの前後に出ている曲線制御点。Graph Editor で操作。
Morph Target / Shape Key頂点位置の差分セット。「笑い顔」「怒り顔」などを複数登録して混合できる。
Modifierメッシュを非破壊的に変形する Stack 操作。Subdivision / Mirror / Array など。
CSG (Constructive Solid Geometry)2 メッシュのブール演算。Union / Subtract / Intersect。
PBR (Physically Based Rendering)物理ベースの質感表現。Albedo (色) / Metallic (金属度) / Roughness (粗さ) などのパラメータで制御。
HDRIHigh Dynamic Range Image。広い輝度範囲を持つ環境マップ画像 (.hdr / .exr)。リアルな反射と照明用。
MatcapMaterial Capture。事前にレンダリングされた球の画像から表面の見た目を決める手抜きシェーディング。
Extrude / 押し出し選択要素を複製して新しい geometry を生やす。最も基本的なモデリング操作。
Inset / インセット面の内側にスケールしたコピーを作る。ボタン形状の起点。
Bevel / 面取り辺を「斜めに削った面」に置換。角を丸める。
Loop Cut / ループカットメッシュをぐるりと一周する辺ループを追加。関節用ループなどに。
Knife / ナイフ画面上に引いた線でメッシュの辺を切断し、面を再分割する操作。カメラと線が成す平面で表裏とも切れる (cut through)。§4.7.1 参照。
AO / Ambient Occlusion周囲のジオメトリに遮られて環境光が届きにくい度合い。へこみ・すき間が暗くなる。Forge3D はレイキャストでテクスチャにベイクし glTF の occlusionTexture として出力。§12.6 参照。
Normal Bake / 法線ベイクハイポリメッシュの表面法線をローポリの接空間ノーマルマップに転写すること。retopo（リトポロジー）ワークフローの中核。§12.7 参照。
Cage / ケージ法線ベイクでレイを飛ばす起点となる、ローポリ表面を法線方向に膨らませた仮想面。Forge3D はメッシュサイズの 5% を自動使用。
Driver / ドライバあるプロパティの値を別のプロパティで自動駆動する仕組み。Forge3D ではボーンの姿勢チャンネル → モーフ influence の線形マッピング (Shape Key Driver)。§12.1.1 参照。
Corrective Shape / 補正シェイプ関節を曲げたときの体積減りや食い込みを直すためのモーフターゲット。ドライバで関節角度に連動させるのが定石。
Flip Diagonal / 対角線フリップ隣接 2 三角形の共有辺を反対側の 2 頂点を結ぶ辺に張り替える操作。§4.7.2 参照。
Shade Smooth / Flat面をなめらか / フラットにシェーディングする法線制御。§4.8.3 参照。
Auto Smooth / 自動スムーズ指定角度より急な折り目だけハードエッジにし、ゆるやかな面はなめらかに繋ぐ法線制御。§4.8.3 参照。
Edge Slide / エッジスライド辺ループを隣の辺 (レール) に沿って横滑りさせる操作。Loop Cut 後のループ位置決めに使う。§4.6.1 参照。
Vertex Slide / 頂点スライド頂点 1 つを隣の頂点との辺に沿って滑らせる操作。トポロジ不変。Blender の Shift+V 相当。§4.6.2 参照。
Proportional Editing / プロポーショナル編集選択の周囲の頂点も減衰付きで一緒に変形する編集モード (soft select)。Blender の O キー相当。§4.2.2 参照。
Merge / Collapse複数の頂点 (または辺の両端) を 1 点にまとめる操作。§4.8.1 参照。
Bridge (Edge Loops)2 本の境界エッジループを面の帯で繋ぐ操作。筒どうしの連結に。§4.8.2 参照。
Seam / シームUV 展開時にメッシュを「切る」境界辺。指定された辺で島が分かれる。Edit Mode の Edge 選択 + Shift+S で指定。
UV Unwrap / UV 展開3D メッシュ表面を 2D 平面に展開すること。テクスチャマッピングの前提。Forge3D は Smart UV Project (法線クラスタリング + シーム尊重) を採用。
UV Island / UV 島UV 空間内で連続した面のひとかたまり。シームや法線断絶で島が区切られる。UV Editor (§4.9.5) で移動 / 回転 / 拡縮できる。
Texture Stretch / ストレッチ3D 上の三角形と UV 上の三角形の形の乖離。大きいとテクスチャが引き伸ばされたり潰れたりして見える。UV Editor のストレッチ表示 (青→赤ヒートマップ) で可視化。全島共通のテクセル密度からのずれも歪みとして扱う。
Pack / パッキングUV 島を 0–1 の UV 空間内に重ならないよう詰めること。Forge3D はシェルフ rect-pack（島を高さ順に棚へ詰め、共通スケールで正規化）。§4.9.3 参照。
LSCM / ABF角度を保存する UV 展開アルゴリズム。Forge3D は V2 で導入予定。V1 は単純な平面投影。
OPFSOrigin Private File System。ブラウザ内でファイルを永続化する Web API。GB 単位の容量に対応。
PWA (Progressive Web App)インストール可能な Web アプリ。オフライン動作・ホーム画面追加など、ネイティブアプリ風の体験を提供。
FXAA / SSAO / Bloomポストプロセス効果。FXAA = 軽量アンチエイリアス、SSAO = 環境光遮蔽、Bloom = 明るい部分の光漏れ。
Auto-Keyポーズ変更 (回転ドラッグ) を終えるたびに、そのボーンのキーフレームを現在フレームへ自動記録するモード。Anim タブの 🔑 トグル。
.forge3dForge3D のポータブルなプロジェクトファイル。シーン GLB + サイドカー JSON (手続きグラフ / Sculpt マスク / レイヤー / ボーン Roll / ボーンコンストレイント / Shape Key Driver) を 1 ファイルに束ねた独自コンテナ。§13.4 参照。

16. モバイル / タッチ操作

Forge3D はタブレット・スマホでも動作する。デスクトップとの差分：

16.1 レイアウト

  • 横幅 900px 以下で モバイルレイアウトに切替
  • 左右のパネルは オーバーレイとして表示 (340px 幅、最大 85vw)
  • ヘッダーの「☰」ボタンで左パネル、「⋮」ボタンで右パネルをトグル
  • 画面下に Mobile Bottom Bar (Prim / Undo / Dup / Del / Export / Save / Load / Cam / Multi / Edit)

16.3 Edit Mode をモバイルで使う

  1. メッシュを選択
  2. Bottom Bar の ✎ Edit ボタンをタップ → Edit Mode に入り、右パネルが自動で開く
  3. 右パネルから Vertex / Edge / Face を選択 (キーボード不要)
  4. キャンバスをタップでコンポーネント選択
    • 選択判定半径はモバイル時 ×2.2 倍に自動拡張 (指で当てやすく)
  5. 右パネルから Extrude / Inset / Bevel / Loop Cut / Knife / Mark Seam / Unwrap / Delete を実行
  6. Gizmo は ×2.5 倍に拡大表示、平面ハンドルも自動有効化
  7. 再度 ✎ Edit ボタンで Object Mode へ戻る

16.4 カメラロック

Bottom Bar の 🔒 Cam でカメラ操作を一時停止できる。 細かい選択作業中にビューが動くのを防ぐのに有用。

16.5 既知の制限

  • Box Select はモバイルでもボタンから起動できるが、キャンバス上のドラッグが カメラ操作と競合することがある → 必要なら先に 🔒 Cam でカメラをロック
  • テンキー カメラプリセット (1/3/7) はモバイルキーボードでは効かない → 必要なら設定パネル経由
  • Edit タブの操作ボタン群は右パネル内で縦スクロールになる場合あり (画面が小さいと長くなる)
タブレットなら 7〜10 インチクラスが快適。スマートフォンでも動くが、頂点選択精度は厳しい。

17. 更新ルール

forge3d のコードを変更したら、このファイル (MANUAL.html) も同じコミット内で更新すること。

forge3d/CLAUDE.md にも同じルールが書かれている。具体的には以下のいずれかが発生したら本書を更新：

  • 新しい キーバインドの追加・変更 → §14 ショートカットと該当機能の説明を更新
  • 新しい 機能 / オペレータの追加 → 該当セクションに追記、必要なら新セクション
  • 既存機能の 挙動変更 (たとえば Bevel の制限緩和、Knife の対応範囲拡大など) → 該当節を更新
  • UI 配置の変更 (新タブ、パネル移動など) → §1.1 と該当節を更新
  • 新しい 用語を導入 → §15 用語解説に追加

コードと文書が乖離するとユーザーが「動くはずの操作が動かない」状態になるので、 小さな変更でも同期させる。
