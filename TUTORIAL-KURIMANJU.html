くりまんじゅうを Forge3D で作る

Blender を経由せずに forge3d だけで 1 体のキャラクターを完成させるための実践ガイド。 くりまんじゅう（chiikawa-reign の主人公キャラ）の構造を例にしてるが、 丸いシルエットのキャラなら手順をそのまま流用できる。
制作時間目安: 90〜120 分（初回）／30〜45 分（2 回目以降）

0 準備

forge3d を立ち上げて、これから作る形のイメージを掴む。

0.1 forge3d を開く

  1. npm run dev -w forge3d で開発サーバー起動 (または公開 URL の forge3d を開く)
  2. ブラウザで forge3d 画面を確認。3 カラム：左パネル / ビューポート / 右パネル
  3. 左パネル「Primitives」が見えてれば OK

0.2 完成形のイメージ

くりまんじゅうの形を分解すると：

╱─╲ ← 耳 ×2 (細長 ellipsoid) ╱ ╲ │ 頭 │ ← 横長 ellipsoid (0.6 × 0.5 × 0.5) ╲___╱ │ │ ← 胴体 (半球、頭の下に) │ │ ┐ ← 尻尾 (小さい ellipsoid) ─┴─┴─ │ │ ← 腕 ×2 (細い cylinder、体側面から) │ │ ┴ ┴ ← 脚 ×2 (短めの cylinder、体底面から)

これを順番に組み立てる。基本は「プリミティブで配置 → 形を整える → 左右対称は Mirror Modifier」。

0.3 単位と尺度

パーツ直径 / 高さ
頭0.6 × 0.5 × 0.5
胴体0.5 × 0.6 × 0.4 (半球)
耳0.08 × 0.15 × 0.06
尻尾0.05 × 0.09 × 0.06
腕高さ 0.20、太さ 0.03
脚高さ 0.16、太さ 0.03

全体で 1m 弱（頭頂〜足先）に収まる。chiikawa-reign の MovementSystem がこのスケール前提。

参考画像を見ながら作りたい場合: Scene タブの 🖼 Import Reference で くりまんじゅうの絵を半透明の下絵プレーンとしてシーンに置ける（マニュアル §2.1.1）。 Gizmo で横に置いてもいいし、正面ビューの奥に立てて三面図的に使ってもいい。 GLB エクスポートには含まれないので置きっぱなしで OK。

1 頭と胴体

2 つの sphere を配置して、扁平させて、頭と胴体の基本形を作る。

1.1 頭を作る

  1. 左パネル「Primitives」→ Sphere をクリック
  2. 原点に sphere（直径 1.0）が生成される。これが「頭」候補
  3. 右パネル Transform タブで以下を設定：
    Position: 0, 0.46, 0 ／ Scale: 0.6, 0.5, 0.5

    頭がやや横長の楕円体になり、地面から少し浮いた位置に配置される。

  4. 右パネル「Material」タブで色を変更：
    • Albedo Color: #FFEAC7 (くりまんじゅう色 = 薄い肌色)
    • Metallic: 0、Roughness: 1
  5. 右パネル「Properties」のメッシュ名を head に変更

1.2 胴体を作る

  1. もう一度 Sphere → 新しい sphere が原点に生成
  2. Transform で：
    Position: 0, 0.06, 0 ／ Scale: 0.5, 0.6, 0.4
  3. 同じ色のマテリアルを設定（Material タブで色を pick、あるいはマテリアルを使い回す）
  4. 名前を torso に

1.3 胴体を半球にする

くりまんじゅうの胴体は「お饅頭の下半分」のような半球形。Edit Mode で上半分を削る：

  1. torso を選択した状態で Tab → Edit Mode に入る
  2. 3 で Face モードに切り替え
  3. カメラを上から見下ろす視点に（Numpad 7）
  4. Box Select B で上半分の面を矩形選択
  5. X で削除（Delete Faces）
  6. Tab で Object Mode に戻る
Edit Mode に入ると右パネルが Edit タブに自動で切り替わる。 モード切替に迷ったらこの色（V/E/F ボタンのハイライト）を見ると現在の選択モードが分かる。
Box Select で「上半分だけ」選びたいのに胴体の裏側まで透けて選んでしまう場合、 カメラを真上ではなく斜め上にして、裏の面が見えない角度から選ぶと正確。 あるいは 1 面ずつクリックで個別選択もアリ。

2 耳 (左右)

細長い ellipsoid を頭に取り付けて、Mirror Modifier で左右対称にする。

2.1 片耳を作る

  1. Sphere を追加
  2. Transform：
    Position: 0.15, 0.7, -0.05
    Rotation: 0, 0, 60° (= π/3)
    Scale: 0.08, 0.15, 0.06
  3. 名前 ear_R（右耳）

2.2 もう片耳は Mirror Modifier で

左耳を別途作るのではなく、Mirror Modifier で右耳を反転コピーする。手動で 2 個作ると左右ズレが起きやすい：

  1. ear_R を選択した状態で右パネル Transform タブ → 「Modifiers」セクション
  2. 「+ Mirror」をクリック
  3. Axis: X、Merge: off
  4. 左耳が即座に出現する（モディファイア = 非破壊なので元データはそのまま）
ear_R の Position の X 座標は 正の値（0.15）。Mirror で X 軸反転されると -0.15 側に複製される。 これでくりまんじゅうの片側だけを動かす（後で耳をピクピクさせたい等）ことも可能。
Mirror で「お腹に向かって耳が裏返って入り込む」場合、Axis 設定がズレてる可能性。 くりまんじゅうの左右対称軸は X。Y/Z の Mirror は前後・上下対称になるので別物。

3 腕と脚

細い cylinder を 4 本作る。Mirror Modifier で左右をペアに。

3.1 腕

  1. Cylinder → 原点に円柱が出る
  2. Transform：
    Position: 0.20, 0.20, 0
    Rotation: 0, 0, 36° (= π/5、腕を少し前に振った角度)
    Scale: 0.03, 0.10, 0.03

    ※ Cylinder の高さは scale.y × 2 になる仕様、Cylinder のデフォルト高さ = 1。 つまり scale.y = 0.10 で実高さ = 0.20。

  3. 名前 arm_R
  4. Mirror Modifier (Axis: X) を追加 → 左腕が自動生成

3.2 脚

  1. Cylinder
  2. Transform：
    Position: 0.08, -0.10, 0
    Rotation: 0, 0, 0
    Scale: 0.03, 0.08, 0.03
  3. 名前 leg_R
  4. Mirror Modifier (Axis: X)

4 尻尾

小さな ellipsoid を 1 個、胴体の後ろに取り付ける。
  1. Sphere
  2. Transform：
    Position: 0, 0.10, -0.25
    Scale: 0.05, 0.09, 0.06
  3. 名前 tail

尻尾は対称じゃないので Mirror Modifier 不要。1 個きり。

5 シルエット調整

「饅頭らしさ」「ぷにっと感」が出るように頭と胴体を整える。Sculpt と Edit Mode の合わせ技。

5.1 頭を「お饅頭」にする — Sculpt

  1. head を選択
  2. D で Sculpt ツールに切替
  3. 右パネルのブラシ設定：
    • Radius: 0.15
    • Strength: 0.05
    • Falloff: 2
  4. 頭頂部を Pull ブラシで少し引っ込めて「ふっくら平たい」感じに
  5. 頬の側面を Inflate でほんのり膨らませる
Sculpt は元に戻せる：ブラシ 1 ストロークごとに Ctrl+Z で巻き戻る。 やりすぎたら戻して再開。

5.2 胴体下部を丸める — Edit Mode で Loop Cut + 移動

胴体の底面を少し丸める：

  1. torso を選択 → Tab
  2. 2 で Edge モード
  3. 胴体の下半分にある水平な辺を 1 本クリック
  4. Ctrl+R → Loop Cut → 自動で水平ループが追加される（中点が選択状態に切替）
  5. 追加されたループ頂点を Gizmo で少し内側 (Y 方向に -0.02 程度) に動かす
  6. Tab で Object Mode

6 クリーンアップ

UV 展開とリギングの前に、メッシュをクリーンな状態に整える。

6.1 法線をきれいに

各メッシュごとに：

  1. メッシュを選択
  2. 右パネル「Mesh Tools」→ Recalc Normals

6.2 隣接頂点をマージ (オプション)

Sculpt で頂点が非常に近接してしまった箇所がある場合：

  1. Mesh Tools → Weld Vertices (許容値はデフォルトの 0.001)
Weld 後は頂点数が減るので、後の Weight Paint に影響する。 リギングの後で weld するとウェイトが乱れるので、必ずリギング前に。

6.3 メッシュ統合の検討

くりまんじゅう本体（頭 / 胴体 / 耳 ×2 / 腕 ×2 / 脚 ×2 / 尻尾 = 計 8〜9 メッシュ）を 1 メッシュに統合するか別個に保つかの選択：

方針メリットデメリット
別個に保つ各パーツの色や材質を個別設定可。動的に隠せるリギング時に skeleton が各メッシュに別々に attach 必要
統合する1 つの skeleton で全身を skinning できる。GLB エクスポートが綺麗1 つのマテリアル前提。後で色を変えづらい

chiikawa-reign 用には 統合方針を推奨。GLB 内で 1 メッシュとして扱える。

6.4 統合する場合

  1. head, torso, ear_R, arm_R, leg_R, tail を全部 Ctrl+クリックで複数選択
  2. Mesh Tools → Merge Meshes（または右クリック → Join）
  3. 統合された 1 メッシュが残る、名前を kurimanjuu に
Merge する前に Mirror Modifier の Apply（確定）が必要。Modifier は非破壊なので、 Apply しないと Mirror で生成された左側がマージ後に消える。Modifier タブで「Apply」ボタンを各 Modifier に。

7 UV 展開

テクスチャを貼るための 2D UV 座標を生成する。顔を描く前にこれをやる。

7.1 シーム指定 (任意)

シームを指定しなくても Smart UV Project は自動で「面の法線」でクラスタリングしてくれる。 ただし、顔の前面を 1 つの島にしたい場合は明示的にシームを切るのがおすすめ：

  1. kurimanjuu を選択 → Tab
  2. 2 で Edge モード
  3. 頭の側面〜後頭部にぐるりと一周する辺を選択（顔と後頭部の境目）
  4. Shift+S または右パネル「Mark Seam」
  5. 選択辺が赤色になる

7.2 UV 展開実行

  1. Edit Mode のまま、右パネル「UV Unwrap」→ Smart UV Project（または U）
  2. ステータスバーに「Unwrap: NNN verts, NNN faces」と表示されれば成功
UV 展開は頂点数を増やす。リギング済みでもボーンウェイトは 自動で引き継がれるようになったが、頂点が増えるぶんウェイトの塗り直し確認は少なく済ませたいので、 リギング前に展開しておくのが引き続きおすすめ。Morph Target 付きは展開不可 （モーフ作成前に展開すること）。
展開結果の 2D ビューはまだ無いので、ペイント時に「あ、ここの UV が伸びてる」と 気付いてから手直しする流れ。Phase 2 で 2D UV エディタを実装予定。

8 顔を描く

UV 展開済みのメッシュにテクスチャを直接ペイントして、目・口・頬を描く。

8.1 ペイントモードに入る

  1. Tab で Object Mode に戻る (Edit Mode のままだと Paint が使えない)
  2. kurimanjuu を選択した状態で P で Paint ツール
  3. 右パネルの Paint タブで設定：
    • Color: #000000 (目と口は黒)
    • Size: 8
    • Opacity: 1.0

8.2 目を描く

  1. 顔の中央付近（カメラ位置 = 正面ビュー Numpad 1）にメッシュを向ける
  2. 右目の位置 (x ≈ +0.10, y ≈ 0.51) に小さなドットをクリックで描く
  3. 左目 (x ≈ -0.10) も同様に
  4. 離れた 2 つのドット = 黒い豆みたいな目になる

8.3 口

  1. Color はそのまま黒
  2. Size を 4 ぐらいに下げる
  3. 目の下、顔の中央に小さな縦線または「く」の字を描く

8.4 頬を描く

  1. Color: #FFA8A8 (ピンクっぽい色)
  2. Size: 12、Opacity: 0.5
  3. 左右の頬の位置 (目の真下、x ≈ ±0.15) にやわらかい円を描く
UV が変な伸び方をして塗れない / 違う場所に描けてしまう場合、手順 7 (UV 展開) に戻って、 別のシーム指定で再展開してみる。 何度でも展開しなおせるので試行錯誤 OK。

9 ボーン入れ (リギング)

アニメーションのためのスケルトンを組む。くりまんじゅうは骨が少なめ。

9.1 必要なボーン構成

root (脚の付け根あたり) ├── spine (体の中心、腰〜首の手前) │ ├── neck → head_bone (頭部の中) │ ├── arm_L_upper → arm_L_lower → hand_L │ ├── arm_R_upper → arm_R_lower → hand_R │ └── tail_bone (尻尾の中) ├── leg_L_upper → leg_L_lower → foot_L └── leg_R_upper → leg_R_lower → foot_R

計 15 本程度。最初は腕と脚それぞれ 1 本ずつでも OK（アニメに必要になったら細分化）。

9.2 スケルトン作成

  1. B で Bone ツール
  2. 右パネル Bone タブの「+ New Skeleton」
  3. 右パネル「Display」で X-Ray ON、Bone Size 1.5〜2.0 ぐらいに調整 （ボーンがメッシュ内側にあっても見えるように、かつ大きすぎず）

9.3 ボーンを配置

  1. カメラを正面ビュー Numpad 1 に
  2. 体の中心、地面ぎりぎりの位置をクリック → root ボーン
  3. root が選択された状態で、上方 (胴体の中央) をクリック → spine (root の子として作成)
  4. spine を選択した状態で頭の中心をクリック → neck → さらに上で head_bone
  5. spine 選択 → 右肩あたりをクリック → arm_R_upper → 肘 → 手で順次（右側だけでOK）
  6. 脚も右側だけ作る
ボーンを作る時に「親を選択した状態で次の位置をクリック」=「親子付きで作成」になる。 親を別のに変えたいときは別のボーンを選び直してからクリック。

9.4 左側はミラーで一発生成

  1. 右腕の根元（arm_R_upper）を選択
  2. 右パネル「Selected Bone」の ⇄ Mirror Bone (X) を押す
  3. arm_R_upper → arm_L_upper として左腕一式が反転生成される（名前の _R が _L にスワップ）
  4. 右脚の根元でも同様にミラー
背骨（spine）など中心軸上のボーンはミラー不要。腕・脚など左右対になるものだけ右で作って ミラーすれば、左右対称が保証されて手早い。子ボーン（肘・手など）も一緒にミラーされる。

9.5 メッシュにアサイン

  1. kurimanjuu メッシュを選択（Object Mode）
  2. 右パネル「Assign to Mesh」
  3. skeleton と mesh の関連付け完了

10 ウェイト塗り

各ボーンが頂点をどれだけ動かすかを決める。

10.1 まず Auto Weights で下地を作る

  1. Bone タブで「Assign to Mesh」がまだなら先に実行（スケルトンをメッシュに割当）
  2. kurimanjuu を選択 (Object Mode)
  3. W で Weight Paint ツール → 右パネル「Weight Paint」の ⚡ Auto Weights を押す
  4. 全頂点が最寄りボーンへ自動バインドされる。これだけで大まかな変形は成立する
くりまんじゅうは骨が少ないので Auto Weights だけでもかなり綺麗に決まる。 以降の手塗りは「気になる箇所の手直し」と考えてよい。Auto Weights は既存ウェイトを上書きするので、 手塗りを始めた後にもう一度押すとリセットされる点に注意。

10.2 Weight Paint モードで確認

  1. W で Weight Paint ツール（既に入っていれば OK）
  2. Show/Hide Overlay でヒートマップ表示にして、ボーンごとの影響範囲を確認

10.3 ボーンを選んで手直し

  1. spine ボーンをクリックで選択 (ボーンビジュアルが見える状態で)
  2. 右パネル「Weight Paint」設定:
    • Radius: 0.10、Strength: 0.5、Mode: Add
  3. 胴体部分のメッシュをドラッグ → 明るくなる箇所 = spine の影響度高い
  4. 同様に neck → 首あたり、head_bone → 頭、arm 各ボーン → 腕

10.4 ウェイトのコツ

  • 関節部分はグラデーション: 肘なら upper と lower で半々ぐらい
  • 全頂点が必ずどれか 1 つ以上のボーンに 100% 影響: 漏れがあると変形時にその頂点が原点に固定される
  • Mode 切替: Add で塗る / Subtract (Ctrl) で削る / Smooth (Shift) でぼかす
塗り忘れチェック: 全ボーンを順に選んで「このボーンに反応する頂点」を確認。 どこか明るい箇所が無い頂点があれば塗り忘れ。

11 ポーズ確認

作ったリグが正しく動くか、Pose Mode で簡単な姿勢を作って確認。
  1. 右パネル Bone タブ → モード切替で Pose を選択
  2. arm_R_upper をクリックで選択
  3. Rotation Gizmo (3 軸の輪っか) が表示される → 各軸を回してみる。 リングはデフォルトでボーン自身の軸に沿う（Mode の「Local 軸回転」ON。 OFF にするとワールド軸。リングの捻り向きは Selected Bone の Roll° で調整可 — マニュアル §7.2.1 / §7.3.1）
  4. 腕が肩を支点に動けば OK。胴体や頭まで一緒に動いてしまったら、ウェイトの影響範囲を見直す
  5. 各ボーンを動かして、メッシュが想定通りに変形するかチェック
変形が「グニャっと崩れる」「変な方向に裂ける」場合は Weight Paint の見直し。 特に隣接する 2 ボーンの境目で重みが急に切り替わると崩れる。Smooth で滑らかに繋ぐ。

ポーズの状態を一旦元に戻す: Edit モードに戻して再選択 (rest pose に戻る)。または各ボーンの rotation を 0 にリセット。

左右対称のポーズは片側だけ作れば OK。arm_L_upper をポーズ → 📋 Copy → 🪞 Mirror で、対側の arm_R_upper に 左右反転したポーズが入る（マニュアル §7.2.2）。
腕や足を「手先の位置」で直感的に動かしたいときは IK が便利 （マニュアル §7.6）。先端ボーン（hand_R 等）を選んで Bone タブの「IK Constraint」で Enable IK → Chain Length 2 → Snap to Bone してから Target を動かすと、肘・肩が自動で曲がる。 肘が裏返るときは Pole (bend dir) を ON にして Snap to Bend → Pole 座標で曲がる向きを固定する。くりまんじゅう程度の単純リグなら必須ではないが、ポーズ付けが一気に楽になる。
関節が「ありえない方向」に曲がるのを根本から防ぎたいときは、Bone タブの Bone Constraints（マニュアル §7.7）。Limit Rotation で肘ボーンの 回転を軸ごとに min/max 度でクランプでき（IK の解にも効く）、Aim (注視) は 頭や目のボーンをターゲット座標へ向け続けてくれる（有効化前に Snap Ahead でポーズ跳び防止）。
表情を動かしたい場合: Morph タブでターゲット（目閉じ・口開け等）を作っておくと、 Anim タブのクリップに モーフキーを打てる（マニュアル §9.2.2）。🔑 Auto-Key ON なら Morph スライダーを動かすだけで現在フレームにキーが入り、GLB にも表情アニメとして出力される。 くりまんじゅうのまばたきはこれで 3 キー（開→閉→開）置くだけ。
歩きモーションなどを作り込むときは: タイミング調整はドープシートの キーをドラッグ（Shift+クリックで複数選択 → まとめて移動、マニュアル §9.3.1）、 動きの緩急確認は 👻 Onion Skin（前=緑 / 次=赤 のゴースト骨格、§9.4.1）が便利。 「接地が 2 フレーム早い」みたいな微調整がキーの打ち直しなしでできる。

12 GLB エクスポート

完成したくりまんじゅうを GLB ファイルとして書き出す。
  1. kurimanjuu メッシュ (skeleton 込み) を選択
  2. 左サイドのモバイル時は Export ボタン、デスクトップではメニュー「File → Export → GLB」
  3. 保存場所とファイル名: kurimanju.glb
  4. ダウンロードが始まる
GLB は単一ファイルで mesh / material / texture / skeleton / animation すべて梱包。 別途 png や json は要らない。

12.1 エクスポート後の確認

外部の GLB ビューア (例: https://gltf-viewer.donmccurdy.com) にドロップして、 形と色とアニメ (もしあれば) が想定通りに見えるかチェック。 forge3d 内で見えてた状態と一致してれば OK。

13 chiikawa-reign に持っていく

エクスポートした GLB を実際のゲームで使う。
  1. kurimanju.glb を chiikawa-reign/src/assets/models/characters/ に上書きコピー
  2. chiikawa-reign 側で npm run dev
  3. くりまんじゅうが新しいメッシュで表示される

13.1 既存メッシュとの差分確認

差し替え後、以下を確認：

  • キャラがちゃんと立っている (床にめり込まない、宙に浮かない)
  • 移動アニメ (歩く・走る・ジャンプ) が動く
  • ボーン名が一致してる場合は既存アニメがそのまま流れる
ボーン名や階層が違うと既存の `AnimationGroup` がターゲット解決できず、 キャラがフリーズしたように見える。chiikawa-reign の character-animation.ts で 期待してるボーン名 (root, spine, head_bone, arm_L_upper, hand_R, ...) と一致させる必要あり。 不一致なら最初に kurimanju.ts の procedural バージョンのボーン名を参考に修正。
